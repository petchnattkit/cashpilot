# Task: Migrate Transactions Table Schema

## User Story
**As** a Developer,
**I want** to link transactions to the new master data tables,
**So I can** enforce referential integrity and support detailed reporting.

## Atomic Scope
- Create a migration file `supabase/migrations/<timestamp>_link_transactions.sql`.
- Add `sku_id` (UUID, Foreign Key) to `transactions`.
- Add `category_id` (UUID, Foreign Key) to `transactions`.
- Migrate existing string data from `sku` and `category` columns to the new tables (if applicble) or allow nullable.
- Remove old string columns `sku` and `category` (OPTIONAL for this task, can be separate cleanup).

## Acceptance Criteria
1. Run `npx supabase migration up`.
2. Verify `transactions` table has foreign keys.
3. **Constraint**: Existing transaction data MUST be preserved.
4. **Constraint**: Services using `transactions` will fail until Task 1.3 (Types) is complete (Sequential dependency).

## Impact Area
- Database Schema (`transactions` table)
