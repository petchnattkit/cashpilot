# Task: Implement Categories Master Data

## User Story
**As** a Business Owner,
**I want** to define Categories (Income/Expense),
**So I can** tag my transactions accurately.

## Atomic Scope
- Update `SettingsPage.tsx` to include "Categories" tab.
- Implement `useCategories` hook (CRUD on `categories` table).
- List existing categories.
- Add "Create Category" form (Name, Type).

## Acceptance Criteria
1. Navigate to `/settings` -> Categories Tab.
2. Create "Office Supplies" (Type: Expense).
3. Verify it appears in list.
4. **Constraint**: Type must be restricted to 'in' | 'out' | 'both'.

## Impact Area
- Frontend Route `/settings`
