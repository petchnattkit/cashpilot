# Task: Create Migration for SKUs and Categories

## User Story
**As** a Developer,
**I want** to create a database schema for `skus` and `categories`,
**So I can** store inventory and transaction classification master data.

## Atomic Scope
- Create a single SQL migration file `supabase/migrations/<timestamp>_create_skus_categories.sql`.
- Define `skus` table: `id` (UUID), `code` (Text, Unique), `name` (Text).
- Define `categories` table: `id` (UUID), `name` (Text), `type` ('in', 'out', 'both').
- Enable/Disable RLS as per MVP policy (Disabled).

## Acceptance Criteria
1. Run `npx supabase migration up` and verify success (Exit Code 0).
2. Verify tables exist using `npx supabase db diff` or `psql` inspection.
3. **Constraint**: Must not modify existing `transactions` table (handled in next task).
4. **Constraint**: No API changes in this task.

## Impact Area
- Database Schema (Public)
