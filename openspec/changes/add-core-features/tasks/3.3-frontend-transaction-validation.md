# Task: Transaction Form Validation & Logic

## User Story
**As** a User,
**I want** the system to prevent me from entering invalid dates or conflicting amounts,
**So I can** trust my financial data.

## Atomic Scope
- Update `TransactionsPage.tsx` handleSave logic.
- Rule 1: `date_out` cannot be before `date_in`.
- Rule 2: Cannot have both `cash_in` > 0 AND `cash_out` > 0 (Exclusive).
- Show error message if validation fails.

## Acceptance Criteria
1. Enter `date_in` = Tomorrow, `date_out` = Today. Click Save. -> Error.
2. Enter `cash_in` = 100, `cash_out` = 100. Click Save. -> Error.
3. **Constraint**: Client-side validation is sufficient for MVP.

## Impact Area
- Frontend Route `/transactions`
